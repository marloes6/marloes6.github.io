5 Beoordeling reproduceerbaarheid artikel

5.1 Gegevens artikel

Voor deze opdracht is gebruik gemaakt van het artikel Timescales of influenza A/H3N2 antibody dynamics:

Kucharski AJ, Lessler J, Cummings DAT, Riley S (2018) Timescales of influenza A/H3N2 antibody dynamics. PLoS Biol 16(8): e2004974. https://doi.org/10.1371/journal.pbio.2004974

De onderzoeksvraag van het artikel is: Hoe kunnen de antilichaamproductie tegen en de infectie met influenza A/H3N2 worden gemodelleerd om de mate van infectie en immuniteit binnen een populatie te bepalen?

Voor dit onderzoek is gebruik gemaakt van twee serologische datasets waarin door middel van hemaglutinatie-inhibitie (HI) en microneutralisatie is getest op verschillende influenza A/H3N2 stammen in China en Vietnam. Aan de hand van deze datasets is een model aangepast en uitgebreid om de antilichaam log titer te schatten na infectie met een bepaalde stam. Dit model kijkt naar korte- en langetermijnreactie, antigene senioriteit en kruisreactiviteit tussen stammen die op elkaar lijken. Het model is vervolgens gebruikt om aan de hand van gesimuleerde influenza attack rates de infectiegeschiedenis met influenza van individuele patiënten te genereren. De aanwezigheid van antilichamen is kort na de infectie hoog en kruisreactief. De langetermijnreactie is gericht tegen specifiekere stammen. Het model zorgt voor betere schattingen van immuniteit en attack rates, en kan worden toegepast in het opzetten van onderzoek, ook met betrekking tot andere pathogenen.

5.3 Reproduceerbaarheid R code

De code is te vinden in https://github.com/adamkucharski/flu-model.

De zoekopdracht ‘Data Availability = R code’ op de PlosOne website levert veel artikelen op die code bevatten die volledig of deels in andere codeertalen dan R is geschreven. Pas na het openen van een aantal artikelen kwam ik bij dit artikel uit dat vrijwel volledig bestaat uit R code.

Beoordeling leesbaarheid code: 4/5

De code in main_model.R zorgt er eerst voor dat directories worden aangemaakt en de benodigde packages worden geladen. Daarna worden de data en source files ingeladen en worden de bestanden in c code verzameld. Het is mij niet helemaal duidelijk waarom het laden van de data in comments staat. Vervolgens worden de verschillende analyses gerund en daarna geplot. In het laatste deel van de code wordt gesimuleerde data gegenereerd en geplot.

De repository kan in zijn geheel van Github worden gekloneerd naar een nieuw Rstudio project, en bevat een README-file waaruit blijkt dat de belangrijkste code zich bevindt in main_model.R en deze zich baseert op drie andere files (load_data_[LOCATION].R, sero_functions.R, posterior_analysis_flu.R). Er bevinden zich echter nog veel andere files in de repository waarover geen uitleg in de README te vinden is. Het bestand main_model.R is goed te begrijpen. Bij ieder onderdeel van de analyse is in een comment weergegeven waarvoor de code dient. Daarnaast is de analyse opgedeeld in meerdere hoofdstukken (data loading, runs, plotting) wat de code overzichtelijk maakt. Het is me niet direct duidelijk welke rol de drie source files hebben in deze code, en of ik de main file ook daadwerkelijk kan runnen zonder eerst iets met de source files te doen (ik ga ervan uit van wel). De code in de posterior_analysis_flu.R file is bovendien erg lang, maar dat zorgt ervoor dat de main file wel overzichtelijk blijft.
